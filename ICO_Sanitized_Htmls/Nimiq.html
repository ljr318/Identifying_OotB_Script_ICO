The Basics

All NIM addresses and the amount they hold are stored in one public list.

The list is updated every minute: All transactions are packaged into a so-called “block”. The block is agreed on and the values on the list are changed accordingly.

These blocks form a chain that documents all transactions that have ever happened – the blockchain.

Who agrees on the blocks?

Who agrees on the blocks?

Everyone can participate in the process of adding a new block. A sort of cryptographic riddle is created automatically locking every block and the transactions in it. The answer can be only found by guessing and checking.

Whoever finds the answer first unlocks the block and adds the transactions to the chain. The found answer is quickly verified by the other participants, allowing everyone to agree on the list’s new balances.

Why do people participate?

Why do people participate?

There is a reward of NIM for finding the answer. It is the only way to create NIM.

This process is called ‘mining’.

Can I participate?

Can I participate?

Yes! With Nimiq, mining is easy. Unlike for example Bitcoin, Nimiq does not require you to download software or to configure hardware. One click and your computer starts ‘mining’. Give it a try.

Start Mining

For a full explanation of how this whole process works, the team highly recommends this YouTube video by ‘3Blue1Brown’.

Who controls the network?

Who controls the network?

The Nimiq blockchain is a decentralized and censorship-resistant network that is not controlled by a single authority. Not even the team, the government or criminals are able to change values or delete accounts.

The mechanics and the process of creating the riddles are automated and run on their own. As an open source project, all of Nimiq’s code is public and can be reviewed here.

What about my login?

What about my login?

Your login information is privately created inside your browser. Nobody but you has access to it.

This is great for security and independence, but please keep in mind that common concepts, like password recovery, don’t exist for Nimiq.

Store your login safely and keep it confidential.

Native to the web

Nimiq is a frictionless payment protocol powered by a browser-based blockchain.

Web-native: It has been designed from the ground up to work directly with the latest web technologies such as JavaScript, WebAssembly, WebRTC, IndexedDB (while providing fallbacks) which means that browsers (and the web applications they run) are first-class citizens in Nimiq.

Nano clients

Nano clients

Since it’s impractical to try to download the complete blockchain history in the browser, Nimiq supports Nano clients which can synchronize with the rest of the network (without sacrificing security) in seconds thanks to Non-Interactive Proofs of Proof-of-Work (NiPoPow):

Schnorr signatures

Schnorr signatures

Nimiq uses the EdDSA (Ed25519, specifically) Schnorr variant which allows for smaller public keys and signature sizes (32 and 64 bytes, respectively) while improving security by being able to produce signatures without requiring a random number generator, which may be broken and thus compromise the private key.

Scriptless scripts and multi-sig

Scriptless scripts and multi-sig

The use of Schnorr also allows for multi-signature schemes which require less on-blockchain data (still only 64 bytes) while improving privacy (multisig signatures and accounts are indistinguishable from regular signatures and accounts). Finally, they also allow for the use of scriptless scripts directly with Nimiq.

Accounts tree

Accounts tree

Another important feature that Nimiq has is that, unlike Bitcoin, the balance of each address is stored in a cryptographic data structure, the Accounts Tree, which allows the generation of mathematical proofs of the balance of a given account for a specific moment in time (i.e. block height) so that Nano clients running in web browsers are able to independently verify the validity of a given transaction.

Hashed Time-Locked Contracts

Hashed Time-Locked Contracts

Finally, Nimiq has native support for Hashed Time-Locked Contracts, which enable a wide variety of different scenarios, such as Atomic Swaps that allow users to exchange crypto-assets between different blockchains in a decentralized way (without the need for a trusted third party) or with payment channels.
